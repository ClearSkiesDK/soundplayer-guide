JavaScript Principles for Seasoned Coders

Understanding JS concepts when coming from other programming languages

1. Global State Management

let currentAudio = null;
  • JavaScript uses let for mutable variables (like Python's regular variables)
  • This acts as a singleton pattern - only one audio can exist at a time
  • null is JavaScript's equivalent of Python's None

2. Function Declaration vs Call

function playSound(soundNumber) { ... }  // Declaration
onclick="playSound(1)"                   // Call with argument
  • Functions are first-class objects (like Python)
  • onclick is an HTML attribute that executes JS when clicked
  • Arguments are passed just like Python function calls

3. Conditional Logic & Type Checking

if (soundNumber === 1) { ... } else { ... }
  • === is strict equality (type + value), unlike Python's ==
  • JavaScript has == (loose) and === (strict) - always prefer ===
  • No elif keyword - use else if

4. DOM Manipulation

document.getElementById('button1').disabled = true;
  • document is the global DOM object (think of it as the HTML tree)
  • getElementById() returns an element object (or null if not found)
  • Properties like disabled can be directly assigned (no getters/setters needed)

5. Object Method Chaining & Web APIs

oscillator.connect(gainNode);
gainNode.connect(audioContext.destination);
  • JavaScript objects have methods just like Python classes
  • Web Audio API provides these objects with specific interfaces
  • Method calls are synchronous unless explicitly async

6. Timing & Callbacks

setTimeout(() => {
    currentAudio = null;
    enableAllButtons();
}, 300);
  • setTimeout is like Python's threading.Timer but simpler
  • Arrow function () => { } is like Python's lambda but more powerful
  • JavaScript is single-threaded with an event loop (like Python's asyncio)

Key Differences from Python:

  • No explicit self - functions can be standalone
  • Dynamic typing but with different coercion rules
  • Semicolons optional but recommended for clarity
  • Event-driven - the browser triggers your functions
  • No imports needed for built-in Web APIs

Python Pattern

# Class-based approach
class SoundPlayer:
    def __init__(self):
        self.current = None
    
    def play(self, sound):
        if self.current:
            self.current.stop()
        # ... rest of logic

JavaScript Pattern

// Function-based approach
let currentAudio = null;

function playSound(soundNumber) {
    if (currentAudio) {
        currentAudio.pause();
    }
    // ... rest of logic
}

The Core Pattern

Maintain global state → Check state → Update state → Schedule cleanup

Very similar to managing shared resources in any language, but with JavaScript's event-driven twist.

Summary

JavaScript in the browser is fundamentally about:

  • Responding to events (clicks, loads, timers)
  • Manipulating the DOM (the HTML structure)
  • Managing asynchronous operations (like Python's asyncio)
  • Working with built-in APIs (no pip install needed!)

Once you understand these core concepts, JavaScript becomes much more predictable and familiar.
